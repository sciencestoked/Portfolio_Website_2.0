Ugo Inc Internship: Autonomous Navigation with Lidar Mapping

During my offline internship at Ugo Inc, I had the opportunity to work on developing and deploying advanced robotic solutions, focusing on autonomous navigation using lidar mapping. The project was divided into two main workspaces: lidar_test for mapping and final_navigation for autonomous navigation.

Project Overview

The goal of the project was to achieve autonomous navigation using lidar mapping. This involved creating a detailed map of the environment using a lidar sensor and then using that map to navigate the robot autonomously. The project structure was organized into various packages, each serving a specific purpose.

Workspace: lidar_test

The lidar_test workspace was primarily focused on mapping. Here are the key packages and their functionalities:

  • hector_slam: Used for mapping. The mapping_default.launch file was modified for mapping with joystick control.
  • joy: Facilitates joystick control for the mapping process.
  • Odometry Calculator: Receives twist messages and calculates odometry, then publishes it.
  • my_package: Receives data from the server and publishes it as twist messages for odometry.
  • ngk_kart_desc: Contains the URDF description of the robot.
  • rplidar_ros: Responsible for launching the RPLidar sensor.
  • twist_to_array: Reads cmd_vel data and converts it into an array for robot movement.

Workspace: final_navigation

The final_navigation workspace was dedicated to achieving autonomous navigation. Here are the key packages and their functionalities:

  • my_odometry_calculator: Similar to the package in the lidar_test workspace, it calculates odometry from twist messages and publishes it.
  • my_package: Receives data from the server and publishes it as twist messages for odometry.
  • ngk_kart_desc: Contains the URDF description of the robot.
  • twist_to_array: Reads cmd_vel data and converts it into an array for robot movement.
  • my_launcher: This is the main navigation stack package consisting of configurations for costmaps, a launch file for running all the nodes required for autonomous navigation, and a saved maps folder. It also contains a script to publish dynamic transformations from the Odom frame to base_footprint.

Running the Autonomous Navigation

To achieve autonomous navigation, the following steps were performed:

  1. Launch the navigation stack: roslaunch my_launcher launch-all.launch
  2. Open Rviz to visualize the navigation components.
  3. Set the initial 2D pose to align the laser scan with the map boundary for an approximate initial position.
  4. Set a 2D point and pose as the final navigation goal.
  5. The robot attempts to navigate to the specified goal.
